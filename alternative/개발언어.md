범위가 큰 프로젝트라 언어만 딱 잘라 말하되, “실제로 되는 네트워크 시뮬레이터” 기준으로 잡겠습니다. Packet Tracer 연동 여부는 핵심 리스크라 최신/공식 쪽만 확인해 보겠습니다.

21s 동안 생각함

언어만 말하면 이렇게 가는 게 맞음.

프론트: TypeScript

웹 UI, 드래그 앤 드롭, 장비 배치, 선 연결, 설정창, 사용자 화면은 TS가 제일 맞음. JS 말고 TS로 가야 나중에 장비 타입, 포트 타입, 링크 타입, 설정값 관리할 때 덜 터짐.

시뮬레이션 엔진: Rust

진짜 패킷 흐름, 라우팅, 스위칭, ACL, NAT, DHCP, ARP 같은 걸 “작동”시키려면 TS만으로는 빡셈. Rust로 엔진 만들고 WebAssembly로 브라우저에서 돌리는 쪽이 좋음. WebAssembly는 웹 안에서 고성능/샌드박스 실행에 쓰이는 구조라 이런 시뮬레이터에 잘 맞음.

백엔드: Go 또는 Rust

사용자 저장, 프로젝트 저장, 계정, 파일 관리, 협업 기능까지 생각하면 백엔드 필요함.

추천은 Go. 이유는 서버 API, 동시 접속, 파일 처리, 배포가 Rust보다 단순함.
근데 팀원이 Rust 잘하면 백엔드도 Rust로 통일해도 됨.

DB: PostgreSQL

언어는 아니지만 저장 기능 때문에 거의 필수. 프로젝트, 노드, 링크, 설정값 저장하기 좋음.

.pkt 연동: C++/Rust 쪽 가능성 있음, 근데 매우 위험

Cisco Packet Tracer는 공식적으로 네트워크/IoT/사이버보안 학습용 시뮬레이션 툴이고, .pkt는 Packet Tracer에서 쓰는 파일 형식임.
문제는 .pkt 파일 포맷이 공개 표준처럼 다루기 쉬운 게 아니라 사실상 proprietary binary 쪽이라, 완전 호환 export/import는 개빡셈. 이 기능을 필수로 박으면 프로젝트 난이도가 몇 배로 뜀.

결론:

Frontend: TypeScript
Simulation Engine: Rust + WebAssembly
Backend: Go
Database: PostgreSQL
CLI/Packet logic parser: Rust

최종 추천 조합은 이거임.

TypeScript + React
Rust + WebAssembly
Go + PostgreSQL

JS/TS만으로 전체를 만들 수는 있는데, “Packet Tracer처럼 실제 작동”까지 가면 TS 단독은 비추천. 화면은 TS, 네트워크 시뮬레이션 핵심은 Rust가 맞음.
